## 一、项目概述

将"星宠契约"从本地单机应用（Capacitor + Electron + IndexedDB）重构为 B/S 架构，用户通过浏览器即可使用。

### 目标设备
- 手机（iOS / Android）
- Windows PC
- 教室智慧黑板（Android）

## 三、视觉设计风格 — FIFA卡 + 卡通宠物（混合风）

### 总体原则
两个风格分区，互相补充不冲突。深色区显"专业酷炫"，亮色区显"可爱温暖"。

### 色彩体系

| 区域 | 背景 | 主色调 | 字体风格 |
|------|------|--------|----------|
| 球员卡 / 雷达图 / 排行榜 | 深色渐变 (#1a1a2e → #16213e) | 金色 #FFD700、银色 #C0C0C0、铜色 #CD7F32 | 粗体数字、FIFA 风格 |
| 宠物展示 / 喂养互动 | 柔和渐变（延续原项目宠物背景色） | 暖色系、粉色、天蓝 | 圆体/卡通字体 |
| 商店（魔法集市） | 暖黄 + 木纹质感 (#3e2f1c → #5c3d2e) | 琥珀 #FF8C00、金 #FFD700 | 手写风格招牌 |

### 各页面风格细则

#### 球员卡（个人页 & 教练总览）
- 深色卡片背景，光泽扫过动画（CSS `linear-gradient` + `@keyframes`）
- 评级边框：综合≥85 金色发光 → 70-84 银色 → <70 铜色
- 六维雷达图：霓虹发光线条，渐变填充，hover 放大
- 数字大号粗体（`font-mono` + 发光 text-shadow）
- 宠物形象在卡片上方，保持原项目 `pet-breathe` 等动画

#### 宠物互动区（喂养/玩耍）
- 延续原项目风格：柔和渐变背景、宠物弹跳/摇摆动画
- 饱食度/心情用 emoji 进度条（🍖🍖🍖○○○ 而非干巴的百分比）
- 喂食成功：食材飞入动画 + 宠物跳动 + 爱心飘出
- 进化瞬间：全屏闪光 + 撒花粒子

#### 商店（魔法集市）
- 背景：深色木质纹理 + 悬挂灯笼 CSS 动画
- 商品卡片：羊皮纸/卷轴质感边框，物品图标微浮动
- 分类标签：木牌吊牌风格（`transform: rotate(-2deg)`）
- 购买成功：金币掉落 + 物品飞入背包动画
- 装备/卸下：宠物身上配饰即时显示/消失

#### 首页宠物卡片网格
- 保留原设计骨架
- 卡片加轻微悬浮阴影 + hover 上浮效果
- 宠物不同阶段有不同光晕颜色（蛋=白、幼崽=绿、少年=蓝、成年=紫、稀有=金）

### 字体
- 中文：系统默认（优先 PingFang SC / Microsoft YaHei）
- 数字/评分：`font-mono`（等宽粗体），大号醒目
- 标题/卡片名：`font-bold` + `tracking-wide`

### 动画规范
- 过渡动画：`transition-all duration-300 ease-out`
- 入场动画：列表项 `stagger` 逐项淡入（延迟 50ms/项）
- 卡片 hover：`transform: translateY(-4px) scale(1.02)`
- 雷达图切换：300ms 动画过渡

## 五、角色与权限

### 系统管理员
- 独立登录页，账号密码
- 创建/启用/停用教练账号
- 查看教练注册情况和使用统计
- 批量导入宠物物种、配饰、背景
- 教练默认 7 天试用期，到期需管理员授权续期

### 教练
- 手机号登录，默认密码为手机号后六位，首次登录后可修改
- 管理学生（增删改查）
- 加减积分、配置积分规则
- **球员能力总览**：查看所有学生的六维雷达图 + FIFA 风格球员卡
- **球员对比**：选中多名学生并排对比雷达图
- 管理商店物品
- 导入数据包
- **权限开关**：控制学生端是否允许操作
- 生成学生快捷链接

### 学生
- 无需密码，大屏展示宠物卡片列表
- 点击自己的卡片进入个人面板
- **权限控制由教练决定**：
  - 教练打开权限 → 可喂食、玩耍、购买、换装
  - 教练关闭权限 → 仅可观看大屏展示（所有宠物、排名），不可操作

## 六、教练操作模式开关

教练后台设置一个总开关，控制学生端行为：

| 模式 | 学生端表现 |
|------|-----------|
| **开放模式（on）** | 学生点击卡片 → 进入个人面板 → 喂食、玩耍、商店购物、换装配饰 |
| **展示模式（off）** | 学生点击卡片 → 查看宠物详情（只读），大屏展示所有宠物卡片 + 排行 |

教练可以随时切换，实时生效。

## 七、教练注册与授权流程

```
教练在登录页点击"注册" → 输入手机号 → 系统创建账号（密码=后六位）
                                                    │
                                                    ▼
                                          自动进入 7 天试用期
                                          教练可正常使用所有功能
                                                    │
                                                    ▼
                                          管理员后台查看教练列表
                                        ┌─ 授权（续期）→ 正式使用
                                        └─ 关闭 → 账号停用
```

### 自注册规则
- 教练在登录页可点击"注册"，输入手机号
- 默认密码为手机号后六位，注册成功后可修改
- 首次注册自动获得 7 天试用期
- 管理员可在后台将试用教练转为正式、延长有效期、或关闭账号

## 八、数据模型

### 新增表
```sql
-- 系统管理员
Admin (id, username, passwordHash, createdAt)

-- 教练（替代原 CoachConfig）
Coach (
  id, phone, passwordHash, name, school,
  isActive, trialUntil, authorizedUntil,
  playerMode: 'open' | 'display',  -- 学生端操作开关
  createdAt, updatedAt
)
```

### 评分体系（三层模型，替代原 ScoreRule）

```
ScoreDimension（评分维度）
┌──────────────────────────┐
│ id, coachId FK           │
│ name (如 "技术"、"洞察")   │
│ icon (emoji)             │
│ sortOrder                │
│ isActive                 │
└──────────┬───────────────┘
           │ 1:N
┌──────────▼───────────────┐
│ ScoreIndicator（评分指标）  │
│ id, dimensionId FK       │
│ name (如 "传接球稳定性")   │
│ criteria (评分标准描述)    │── 教练参考，不强制
│ defaultPoints (单次积分)  │
│ dailyLimit (单日上限)     │
│ isActive, sortOrder      │
└──────────────────────────┘

ScoreRecord（扩展）
┌──────────────────────────┐
│ + indicatorId FK (可选)   │── 关联到具体指标
│ + sessionId FK (可选)     │── 关联到训练课次
│ ...原有 playerId, points  │
│     type, reason 保留     │
└──────────────────────────┘

BonusRule（额外奖励）
┌──────────────────────────┐
│ id, coachId FK           │
│ name (如 "全勤奖")        │
│ points                   │
│ frequency: weekly|monthly│
│ criteria (达成条件描述)   │
│ isActive                 │
└──────────────────────────┘
```

系统预置 **「足球青训」评分模板**（6 维度 24 指标 + 5 奖励），教练注册后自动加载。教练可修改/禁用/新增维度和指标。

### 变更表（增加 coachId 外键）
```
Player        ← coachId FK
Pet           ← (通过 Player 间接关联 coach)
ScoreRecord   ← coachId FK
ShopItem      ← coachId FK (coachId=null 表示全局商品)
PlayerInventory ← (通过 Player 间接关联)
```

### 全局配置表（管理员管理）
```
PetSpecies       (id, name, category, stages JSON, ...)
AccessoryDef     (id, name, slot, emoji, position JSON, ...)
PetBackgroundDef (id, name, cssGradient, ...)
ScoringTemplate  (id, name, category, dimensions JSON)  -- 预置模板
```

## 十二、教练端球员能力总览

教练后台可查看所有学生的 FIFA 球员卡和雷达图。

## 十四、与原项目代码复用策略

| 模块 | 处理 |
|------|------|
| 类型定义（types/index.ts）| → `shared/types.ts`，前后端共用 |
| 密码哈希（usePasswordHash.ts）| → `shared/auth.ts`，两端共用 |
| 宠物成长/衰减逻辑 | → `server/src/services/pet.ts`，后端运算 |
| Vue 组件（宠物卡片、展示等）| → 复制到 `client/`，改 `db.*` 为 `api.*` |
| Tailwind 样式 | 直接复用 |
| 配置数据（宠物/配饰/背景）| 移入 PostgreSQL |
| Pinia stores | 改造为 API 驱动的 store |

## 十五、部署

### 环境要求
- 腾讯云服务器（2核4G 起步）
- 已安装 Docker & Docker Compose

### 部署步骤
```bash
# 1. 构建
docker compose build

# 2. 启动（PostgreSQL + API + Nginx）
docker compose up -d

# 3. 初始化数据库
docker compose exec server npx prisma migrate deploy
```

### docker-compose.yml 服务编排
```
services:
  postgres:    # PostgreSQL 数据库
  server:      # Node.js API（端口 3000）
  nginx:       # 前端 + 反代 + HTTPS（端口 80/443）
```
